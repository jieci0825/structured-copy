# Structured Copy MVP 文档

## 1. 背景与目标

当前用户在 `VS Code` 中与 AI、聊天工具、文档或评审场景协作时，经常需要让接收方只关注某个文件中的一小段代码。直接引用整个文件会带来范围不精确的问题，而手动复制代码片段再补充路径、行号说明又增加了额外整理成本。

本轮 MVP 的目标是提供一个通用的结构化复制能力，让用户在不改变默认复制习惯的前提下，可以一键复制出“代码片段 + 文件定位信息 + 行列范围 + 语言信息”的结构化内容，并直接粘贴到 AI、聊天工具、文档或其他引用场景中使用。

## 2. 范围说明

### 2.1 本次要做

- 提供统一命令 `structuredCopy.copySelection`，用于执行结构化复制。
- 提供 3 个触发入口：
  - 编辑器右键菜单 `Structured Copy`
  - 独立快捷键 `Structured Copy`
  - 选区稳定默认约 `200ms` 后显示的 `CodeLens` 入口 `Structured Copy`
- 仅处理当前活动编辑器的主选区。
- 将选区内容格式化后写入系统剪贴板。
- 复制内容包含：
  - 文件路径
  - 选区范围
  - 语言标识
  - 原始代码片段
- 路径优先使用工作区相对路径：
  - 单工作区：使用相对路径
  - 多工作区：使用 `工作区名/相对路径`
  - 非工作区文件：降级为绝对路径
- 范围使用 `1-based` 行列号展示，直接基于 `VS Code` 选区的 `start/end` 值输出。
- 语言标识优先使用编辑器当前 `languageId`，并支持少量常见别名映射：
  - `typescriptreact -> tsx`
  - `javascriptreact -> jsx`
  - `shellscript -> bash`
  - 无法识别时降级为 `text`
- 复制成功后不弹出额外提示

### 2.2 本次不做

- 不改造系统默认 `Cmd/Ctrl + C` 行为。
- 不实现真实悬浮按钮，仅使用 `CodeLens` 作为近似交互。
- 不支持多选区聚合。
- 不支持跨多次复制累积多个片段。
- 不支持多文件片段收集器。
- 不支持自定义复制模板。
- 不支持复制前预览面板。
- 不提供复杂配置项，如绝对路径/相对路径切换、行号样式切换等。

## 3. 核心流程

1. 用户在活动编辑器中选中一段非空代码。
2. 用户通过右键菜单、快捷键或 `CodeLens` 触发 `Structured Copy`。
3. 扩展读取主选区内容、文件路径、范围和语言标识，并生成统一格式文本。
4. 扩展将结果写入系统剪贴板，不额外弹出成功提示。
5. 当选区变化、清空、编辑器切换或失焦时，`CodeLens` 自动隐藏。

## 4. 功能清单

- 功能 1：统一结构化复制命令
  - 提供唯一业务命令 `structuredCopy.copySelection`
  - 所有入口统一调用该命令，避免多处维护复制逻辑

- 功能 2：编辑器右键菜单入口
  - 仅在编辑器中且存在非空选区时显示 `Structured Copy`
  - 触发后执行结构化复制

- 功能 3：独立快捷键入口
  - 为 `Structured Copy` 绑定默认快捷键
  - 不覆盖用户原有默认复制行为

- 功能 4：`CodeLens` 延时入口
  - 监听活动编辑器主选区变化
  - 选区稳定默认约 `200ms` 后显示 `Structured Copy`
  - 在选区变化、清空、切换编辑器或失焦时移除

- 功能 5：统一复制格式化
  - 输出格式固定为：

````text
[file] path/to/file.ts
[range] 12:1-28:2
[lang] ts

```ts
// selected code
```
````

- 功能 6：异常兜底
  - 空选区时命令不可用或给出轻量失败提示
  - 路径无法相对化时降级为绝对路径
  - 语言无法识别时降级为 `text`

## 5. 验收标准

- 在任意文本编辑器文件中选中代码后，可通过右键菜单执行 `Structured Copy`
- 在任意文本编辑器文件中选中代码后，可通过快捷键执行 `Structured Copy`
- 选中代码后停留默认约 `200ms`，可在选区首行上方看到 `Structured Copy` 的 `CodeLens`
- 触发任一入口后，系统剪贴板内容符合统一格式
- 剪贴板内容必须包含 `[file]`、`[range]`、`[lang]` 和代码块
- 普通复制行为保持不变
- 未选中代码时不显示 `CodeLens`
- 快速拖选时不会出现明显闪烁或残留 `CodeLens`
- 切换编辑器、清空选区或失焦后，`CodeLens` 能正确消失
- 非工作区文件也能完成复制，并输出降级后的路径
- 未识别语言的文件也能完成复制，并输出 `text`

## 6. 风险与约束

- 最大风险在于 `CodeLens` 的刷新频率和状态管理，若防抖和状态比较处理不当，容易造成闪烁或残留。
- `VS Code` 扩展 API 不适合实现真实选区浮层，因此本期必须接受 `CodeLens` 作为近似方案。
- 首版只处理主选区，避免过早引入多选区和多片段的数据组织复杂度。
- 为保证 MVP 可控，本期不提前为未来配置化和多片段能力做重抽象，只保留最小闭环实现。
