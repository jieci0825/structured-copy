# Structured Copy 下一阶段 Roadmap

这份清单用于承接 `step6` 之后的迭代方向，帮助后续开发在不打散 MVP 边界的前提下继续演进。

## 迭代原则

- 先扩“复制体验”，再扩“片段管理”。
- 优先做低心智负担、能直接提高日常使用频率的能力。
- 避免在没有真实使用反馈前，过早引入重配置和复杂状态模型。

## P1：高优先级候选项

### 1. 命令失败反馈优化

目标：

在空选区、无活动编辑器、非支持文档等场景下，继续打磨失败提示的一致性与可理解性。

建议内容：

- 统一轻量提示文案
- 区分“不可执行”与“执行失败”
- 补充对应场景的自动化回归

### 2. 手动 UI 验收沉淀

目标：

把当前依赖人工执行的 `Extension Development Host` 验收步骤沉淀成固定 checklist。

建议内容：

- 为三种入口整理标准演示流程
- 为多工作区、非工作区、未知语言补充截图或录屏说明
- 为 `CodeLens` 稳定性补充观察项

### 3. 语言映射扩展

目标：

在不增加配置复杂度的前提下，补充更多高频语言别名映射。

建议内容：

- 根据真实使用频率补充映射表
- 记录哪些语言保持原始 `languageId`
- 为新增映射补充回归样例

## P2：中优先级候选项

### 4. 多选区聚合复制

目标：

支持一次复制多个选区内容，并定义稳定的拼接格式。

设计关注点：

- 片段顺序如何定义
- 范围信息如何展示
- 空选区与混合选区如何处理

### 5. 多片段收集器

目标：

支持跨多次复制累积多个片段，再一次性导出。

设计关注点：

- 数据存储位置
- 片段去重与排序
- 清空、覆盖、追加等交互规则

### 6. 复制模板扩展

目标：

允许在固定输出格式之外，提供少量模板变体。

设计关注点：

- 模板数量必须受控
- 避免把 MVP 拉成配置中心
- 模板变更后如何保证剪贴板输出可预测

## P3：低优先级探索项

### 7. 默认复制行为整合

目标：

探索是否需要把结构化复制进一步融入用户默认复制流程。

风险提醒：

- 容易破坏用户已有肌肉记忆
- 容易与原生复制、其他扩展或系统快捷键冲突
- 明显超出当前 MVP 的保守范围

### 8. 复制前预览

目标：

在少数需要确认格式的场景中提供预览能力。

风险提醒：

- 会拉长操作链路
- 可能削弱当前 MVP 的“一键完成”价值

## 推荐推进顺序

1. 先补齐手动 UI 验收沉淀与失败反馈优化。
2. 再评估是否需要扩语言映射。
3. 有真实使用反馈后，再进入多选区和多片段能力设计。
4. 默认复制整合和预览面板放在后续探索阶段，不进入下一轮 MVP。
