# Structured Copy 执行步骤计划

## 1. 文档目标

基于 [Structured Copy MVP 文档](./structured-copy-mvp.md)，将需求进一步拆解为一份可直接执行的落地计划，帮助项目按“先闭环主链路，再补高风险交互”的顺序推进。

## 2. 执行策略

### 2.1 为什么这样拆

- 先完成“选区读取 -> 结构化格式化 -> 写入剪贴板”的主链路，确保产品核心价值最早可验证。
- 右键菜单和快捷键属于低风险入口，应尽早接入，用来快速验证命令是否稳定。
- `CodeLens` 是 MVP 中风险最高的部分，依赖选区监听、防抖和状态刷新，适合放在核心链路稳定之后实现。
- 异常兜底和验收测试放在后半段集中处理，可以减少前期反复返工。

### 2.2 实施原则

- 所有入口统一调用 `structuredCopy.copySelection`，避免复制逻辑分散。
- 首版只处理当前活动编辑器的主选区，不提前为多选区做重抽象。
- 先保证输出格式正确，再优化交互体验。
- 每完成一个阶段，都要留下可验证产物，确保可以单独验收。

## 3. 建议目录规划

由于当前仓库还没有扩展代码，建议按以下最小结构启动：

```text
package.json
src/
  extension.ts
  commands/
    copy-selection.ts
  core/
    format-selection.ts
    resolve-file-path.ts
    format-range.ts
    normalize-language.ts
  codelens/
    structured-copy-code-lens-provider.ts
  shared/
    editor-guards.ts
```

说明：

- `commands/` 负责命令编排，不直接堆叠格式细节。
- `core/` 放可复用的纯逻辑，便于后续补单测。
- `codelens/` 单独管理状态和刷新逻辑，避免污染主命令。

## 4. 执行步骤

## Step 0：初始化扩展工程

### 目标

建立可运行的 `VS Code` 扩展基础工程，并预留 MVP 所需命令、菜单、快捷键和 `CodeLens` 注册位。

### 任务

- 初始化 `VS Code` 扩展脚手架。
- 配置 `package.json` 中的命令、菜单、快捷键、激活事件和 `CodeLens` 能力声明。
- 建立基础目录结构与入口文件。
- 准备本地调试方式，确保可以在 Extension Development Host 中运行。

### 产出

- 可启动的扩展工程。
- 已注册但尚未完善逻辑的 `structuredCopy.copySelection` 命令。
- 基础文件结构和开发说明。

### 完成标准

- 扩展可以正常启动。
- 命令可以被 `Command Palette` 搜索到。
- 后续实现无需再调整工程基础结构。

## Step 1：实现结构化复制核心链路

### 目标

完成 MVP 最重要的复制闭环，让命令在主选区存在时可将结构化内容写入系统剪贴板。

### 任务

- 读取当前活动编辑器和主选区。
- 校验是否为非空选区。
- 读取选中文本、文件路径、选区范围、语言标识。
- 实现以下核心格式化逻辑：
  - 工作区相对路径计算
  - 多工作区路径前缀处理
  - 非工作区绝对路径降级
  - `1-based` 行列范围输出
  - 常见语言别名映射
  - 未识别语言降级为 `text`
- 组装固定输出格式并写入系统剪贴板。
- 成功后不额外弹出提示。

### 产出

- 可复用的格式化函数。
- 可执行的复制命令。
- 一组典型输入输出样例，方便后续回归验证。

### 完成标准

- 在任意已打开文件中选中非空内容后，执行命令可复制出完整结构化文本。
- 输出必须包含 `[file]`、`[range]`、`[lang]` 和代码块。
- 非工作区文件、未知语言文件都能正常降级。

## Step 2：接入右键菜单与快捷键

### 目标

先完成两种低风险入口，让用户在不依赖 `CodeLens` 的情况下就能使用 MVP 能力。

### 任务

- 在编辑器右键菜单中增加 `Structured Copy`。
- 通过 `when` 条件限制菜单只在编辑器且存在非空选区时显示。
- 配置默认快捷键：
  - macOS：`cmd+alt+c`
  - Windows / Linux：`ctrl+alt+c`
- 确认快捷键不会覆盖普通复制行为。

### 产出

- 菜单入口。
- 默认快捷键入口。

### 完成标准

- 用户可通过右键菜单触发结构化复制。
- 用户可通过默认快捷键触发结构化复制。
- 未选中内容时，菜单不会错误暴露，快捷键不会产生异常行为。

## Step 3：实现 CodeLens 延时入口

### 目标

补上 MVP 中体验最接近“选中即用”的入口，同时控制闪烁和残留风险。

### 任务

- 实现 `CodeLensProvider`。
- 监听活动编辑器变化、选区变化、窗口焦点变化。
- 为主选区变化增加默认约 `200ms` 防抖。
- 仅在以下条件同时满足时显示 `CodeLens`：
  - 当前有活动编辑器
  - 主选区非空
  - 选区在防抖时间内保持稳定
- 将 `CodeLens` 渲染在选区首行上方，并绑定到 `structuredCopy.copySelection`。
- 在以下场景中及时清理显示状态：
  - 选区变化
  - 选区清空
  - 编辑器切换
  - 失焦

### 产出

- 可动态刷新的 `CodeLens` 入口。
- 一套选区状态管理逻辑。

### 完成标准

- 选中代码停留默认约 `200ms` 后可看到 `Structured Copy`。
- 快速拖选时不会出现明显闪烁。
- 清空选区、切换编辑器或失焦后，`CodeLens` 能正确消失。

## Step 4：补齐异常兜底与状态治理

### 目标

解决 MVP 中最容易导致体验不稳定的边界条件，保证命令和 `CodeLens` 在异常场景下可预测。

### 任务

- 为空选区增加命令保护。
- 为无活动编辑器场景增加短路返回。
- 为路径解析失败、语言识别失败增加兜底逻辑。
- 校验 `CodeLens` 刷新是否存在重复触发、状态脏写或残留显示问题。
- 收敛重复的编辑器/选区判空逻辑，避免多处维护。

### 产出

- 更稳定的错误处理逻辑。
- 状态判断辅助方法。

### 完成标准

- 常见异常场景下无报错、无无效复制、无明显 UI 残留。
- 核心逻辑的边界处理规则已固定，不再依赖人工判断。

## Step 5：执行手动验收与回归验证

### 目标

严格对照 MVP 验收标准完成一次完整自测，确认交付闭环成立。

### 任务

- 按 MVP 验收标准逐条执行手动验证。
- 重点覆盖以下场景：
  - 普通文本文件
  - `ts` / `tsx` / `js` / `jsx` / `shellscript`
  - 未识别语言文件
  - 工作区文件
  - 多工作区文件
  - 非工作区文件
  - 快速拖选
  - 编辑器切换与失焦
- 对比实际剪贴板内容与目标格式，确认字段完整且范围正确。
- 记录问题清单并完成最后一轮修正。

### 产出

- 一份验收记录。
- 修正后的稳定版 MVP。

### 完成标准

- MVP 文档中的验收项全部通过。
- 不存在阻断发布的已知问题。

## Step 6：补充交付文档与发布准备

### 目标

让 MVP 具备最小可演示、可交接、可继续迭代的状态。

### 任务

- 补充 README 中的能力说明、使用方式和限制说明。
- 补充快捷键、右键菜单、`CodeLens` 三种入口说明。
- 明确当前不支持的内容：
  - 默认复制行为改造
  - 多选区聚合
  - 多片段收集
  - 自定义模板
- 整理下一阶段迭代候选项，便于后续进入 Roadmap。

### 产出

- 最小交付说明文档。
- 下一阶段待办清单。

### 完成标准

- 新成员只看文档即可理解 MVP 做了什么、怎么用、边界在哪里。

## 5. 建议执行顺序

建议严格按以下顺序推进：

1. Step 0：先把工程和命令注册搭好。
2. Step 1：优先打通结构化复制主链路。
3. Step 2：尽快接入右键菜单和快捷键，获得最早可用版本。
4. Step 3：在主链路稳定后再处理 `CodeLens`。
5. Step 4：集中补边界和状态问题。
6. Step 5：对照验收标准完整回归。
7. Step 6：补文档并整理后续迭代入口。

## 6. 里程碑建议

### M1：核心能力可用

- 命令可执行
- 复制格式正确
- 剪贴板写入成功

### M2：MVP 入口齐备

- 右键菜单可用
- 快捷键可用
- `CodeLens` 可用

### M3：MVP 可交付

- 异常兜底完成
- 验收项全部通过
- 文档补齐

## 7. 当前阶段的重点风险

- `CodeLens` 刷新时机不稳，可能导致闪烁或残留。
- 选区范围使用 `start/end` 直接输出时，若对 `VS Code` 结束列语义理解不一致，容易引起展示偏差。
- 多工作区路径拼接若处理不一致，可能导致输出路径不稳定。
- 若命令逻辑和入口逻辑耦合过深，后续扩展多片段能力时会增加重构成本。

## 8. 建议的下一步动作

如果按这份计划开始实施，第一轮最适合立即落地的是：

1. 初始化扩展脚手架。
2. 注册 `structuredCopy.copySelection` 命令。
3. 完成固定格式文本生成与剪贴板写入。
4. 用右键菜单先验证一次完整闭环。
