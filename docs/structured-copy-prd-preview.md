# Structured Copy 扩展 PRD（预览版）

## 一、背景与问题

当前在 `VS Code` 中协作编码、编写文档，或与 AI / 聊天工具交流时，用户经常需要把某一段代码片段连同定位信息一起精确引用出去。

如果引用范围是整个文件，现有方式已经足够直接；但当范围缩小为：

- 单个文件中的一段代码片段
- 多个文件中的多个代码片段

现有交互会带来以下问题：

- 用户需要先引用文件，再手动复制片段补充说明
- 范围表达不够精确，容易让接收方理解成“整个文件”
- 在多片段场景下，整理上下文成本明显升高

本需求希望提供一种“结构化复制方式”，让用户在 `VS Code` 中选中代码后，能以更低成本复制出“带定位信息的代码片段”。

---

## 二、目标

### 2.1 核心目标

提供一套不影响用户原有复制习惯的结构化复制能力，使用户可以在 `VS Code` 中快速将“代码片段 + 文件定位信息”复制为可直接引用的结构化内容。

### 2.2 目标结果

- 用户仍然保留原有 `Cmd/Ctrl + C` 复制习惯
- 用户可以通过额外交互，快速执行 `Structured Copy`
- 粘贴到目标工具的内容中，包含足够稳定的定位信息
- 支持单文件单片段场景，并为未来多片段能力预留扩展空间

### 2.3 非目标

- 不改造系统默认复制行为
- 不接管普通复制到他人聊天工具时的内容
- 不实现标准编辑器中的真实浮动按钮
- 不在第一阶段处理“跨多次复制累积多个片段”的复杂工作流

---

## 三、产品定位

这是一个 `VS Code` 扩展能力，定位为：

**为代码片段引用场景提供精确上下文复制入口的轻量结构化复制工具。**

它不是通用剪贴板工具，也不是代码分享工具，而是一个面向“代码片段引用”场景的定向辅助能力。

---

## 四、用户画像与典型场景

### 4.1 用户画像

- 日常在 `VS Code` 中浏览、编写、定位代码
- 会在 AI 助手、聊天工具、文档或评审场景中引用代码片段
- 经常需要“让接收方只关注这几行”

### 4.2 典型场景

#### 场景 A：单段问题定位

用户选中一个函数中的几行逻辑，希望让接收方只分析这一段代码的行为与问题。

#### 场景 B：局部改造说明

用户只想指定某个组件中的模板片段或某个函数中的分支逻辑，而不想引用整个文件。

#### 场景 C：多文件关联讨论

用户需要从多个文件中摘取少量关键片段用于引用，后续可扩展为“多片段复制”能力。

---

## 五、需求概述

本次需求包含 3 种交互入口：

1. 选中代码后停留默认约 `200ms`，在选区首行上方显示 `CodeLens` 入口：`Structured Copy`
2. 在编辑器右键菜单中提供 `Structured Copy`
3. 提供独立快捷键触发 `Structured Copy`

用户触发任一入口后，扩展将当前选区转换为约定格式并写入系统剪贴板。

---

## 六、交互方案

### 6.1 方案一：CodeLens 入口

#### 设计说明

用户在编辑器中选中一段非空代码后，若选区在默认 `200ms` 内未发生变化，则在选区首行上方显示一条 `CodeLens`：

```text
Structured Copy
```

点击后执行结构化复制。

#### 说明

这里不是实现“真实悬浮按钮”，而是采用 `VS Code` 标准编辑器能力中的 `CodeLens` 作为近似交互。

#### 显示规则

- 仅在活动编辑器中显示
- 仅当存在非空选区时显示
- 默认只响应主选区
- 当选区变化、编辑器切换、失焦或选区清空时隐藏

#### 价值

- 接近“选中后就地出现操作入口”的体验
- 不改变用户复制习惯
- 学习成本低，视觉上比右键菜单更直接

#### 复杂度评估

**中**

原因：

- 需要监听选区变化并做防抖
- 需要动态刷新 `CodeLens`
- 需要控制显示时机，避免频繁闪烁

---

### 6.2 方案二：右键菜单入口

#### 设计说明

用户选中代码后，在编辑器右键菜单中出现：

```text
Structured Copy
```

点击后执行结构化复制。

#### 显示规则

- 仅在编辑器中显示
- 仅当存在非空选区时显示

#### 价值

- 符合 VS Code 原生习惯
- 最稳定、最直观、最容易实现

#### 复杂度评估

**低**

---

### 6.3 方案三：独立快捷键入口

#### 设计说明

扩展提供单独命令 `Structured Copy`，并绑定一个默认快捷键。

推荐默认值：

- macOS：`cmd+shift+alt+c`
- Windows / Linux：`ctrl+shift+alt+c`

最终快捷键允许用户在 VS Code 中自行修改。

#### 设计原则

- 不覆盖系统默认复制
- 不影响用户既有的 `Cmd/Ctrl + C`
- 优先保证“不会误触”

#### 价值

- 对高频用户最友好
- 效率最高

#### 复杂度评估

**低**

---

## 七、复制内容格式

### 7.1 设计原则

复制结果既要便于工具理解，又要让用户肉眼可读。

格式应满足：

- 带文件定位信息
- 带行号范围
- 带代码语言类型
- 保留原始代码片段

### 7.2 推荐格式

````text
[file] apps/server/src/config/database.ts
[range] 12:1-28:2
[lang] ts

```ts
export const pingDatabase = async () => {
  await database.execute(sql`select 1 as ok`)
  logger.info('数据库连接已就绪', {
    ok: true,
  })
}
```
````

### 7.3 多片段扩展格式

为未来支持多片段复制，格式天然允许多个区块顺序拼接：

````text
[file] path/a.ts
[range] 10:1-18:2
```ts
...
```

[file] path/b.ts
[range] 22:1-35:4
```ts
...
```
````

## 八、功能需求

### 8.1 命令定义

扩展至少提供以下命令：

| 命令 ID | 名称 | 用途 |
| ---- | ---- | ---- |
| `structuredCopy.copySelection` | Structured Copy | 将当前选区复制为带定位信息的结构化格式 |

命名约定如下：

- 产品名：`Structured Copy`
- 命令 / 菜单 / CodeLens 文案：`Structured Copy`
- 命令 ID：`structuredCopy.copySelection`
- 成功后不额外弹出提示

### 8.2 选区处理

- 仅处理当前活动编辑器
- 默认读取主选区
- 若选区为空，则命令不可用或提示失败
- 行列号采用 `1-based` 展示给用户
- 文件路径优先使用工作区相对路径

### 8.3 剪贴板写入

- 执行命令后，将格式化结果写入系统剪贴板
- 复制成功后不弹出打断式提示

### 8.4 语言识别

- 语言标识优先使用编辑器当前 language id
- 若无法识别，则降级为空或 `text`

### 8.5 CodeLens 触发逻辑

- 监听选区变化
- 对选区变化做默认 `200ms` 防抖
- 防抖结束后，若仍是同一活动编辑器的同一非空主选区，则显示 `CodeLens`
- 当任一条件失效时移除 `CodeLens`

## 九、体验细节要求

### 9.1 可预期

用户必须清楚区分：

- 普通复制：仍然是纯代码
- Structured Copy：复制的是带定位信息的结构化内容

### 9.2 轻量

- 不能因为监听选区而造成明显卡顿
- 不能频繁闪烁或污染编辑器视觉
- CodeLens 仅在有明确选区且短暂停留后出现

### 9.3 低打扰

- 不在没有选区时显示任何入口
- 不弹出重提示框
- 复制成功后不应出现打断式弹窗

## 十、技术实现评估

### 10.1 可行性结论

本方案整体可行，且适合作为独立 VS Code 扩展实现。

其中：

- `CodeLens`：可行，适合作为“近似悬浮入口”
- `右键菜单`：可行且稳定
- `快捷键`：可行且稳定

### 10.2 技术边界

标准 `VS Code` 文本编辑器扩展 API 不适合实现“悬浮在选区上方的真实按钮浮层”，因此本方案以 `CodeLens` 替代。

### 10.3 实现复杂度

| 模块 | 复杂度 | 说明 |
| ---- | ---- | ---- |
| 剪贴板格式化复制 | 低 | 命令执行与文本拼接 |
| 右键菜单接入 | 低 | 标准菜单贡献能力 |
| 快捷键接入 | 低 | 标准命令绑定 |
| CodeLens 动态显示 | 中 | 需要选区监听与状态控制 |
| 多片段复制 | 中 | 需要定义收集与拼接机制 |
| 真实浮动按钮 | 高 | 不建议纳入当前方案 |

## 十一、版本规划

### 11.1 MVP

首版建议只做以下能力：

- `Structured Copy` 命令
- 右键菜单入口
- 独立快捷键入口
- 单选区复制
- 统一结构化格式输出
- CodeLens 延时入口

### 11.2 二期

可考虑追加：

- 多选区复制
- 多文件片段收集器
- 自定义复制模板
- 是否包含绝对路径 / 相对路径配置
- 是否包含行号前缀配置
- 复制后预览面板

---

## 十二、验收标准

### 12.1 功能验收

- 在任意文本编辑器文件中选中代码，可通过右键菜单执行 `Structured Copy`
- 在任意文本编辑器文件中选中代码，可通过快捷键执行 `Structured Copy`
- 选中代码后停留默认约 `200ms`，可在选区首行上方看到 `Structured Copy` 的 `CodeLens`
- 触发后，系统剪贴板中内容符合约定格式
- 剪贴板内容包含：文件路径、范围、语言、代码块

### 12.2 体验验收

- 普通复制行为不受影响
- 未选择代码时不显示 `CodeLens`
- 快速拖选时不会出现明显闪烁
- 在切换编辑器或清空选区后，`CodeLens` 可正确消失

---

## 十三、风险与注意事项

### 13.1 CodeLens 刷新频率

若实现不当，可能导致选区变化时频繁刷新，影响体验。需要通过防抖和状态比较减少无效刷新。

### 13.2 多光标与多选区

首版若支持全部多选区，交互和格式复杂度会上升。建议首版只处理主选区。

### 13.3 路径展示一致性

若用户在多工作区或非工作区文件中使用扩展，需要明确路径的降级策略。

### 13.4 语言标识兼容性

个别文件类型可能没有稳定的 language id，需要做好兜底。

---

## 十四、成功指标

可通过以下方式衡量需求价值：

- 用户能在 1 步内完成当前选区的结构化复制
- 相比“先引用文件再补片段”的方式，操作步骤明显减少
- 用户在 AI、聊天工具或文档中对片段范围的表达更精确
- 高频用户能形成快捷键使用习惯

---

## 十五、结论

这是一项**可行、边界清晰、投入适中、收益明确**的扩展需求。

从产品投入产出比来看，推荐采用以下交付顺序：

1. 右键菜单
2. 快捷键
3. CodeLens 延时入口

其中 `CodeLens` 方案已经能较好接近“选中后就地出现操作入口”的体验，足以替代高复杂度的真实浮动按钮方案，适合作为本需求的正式产品方向。
